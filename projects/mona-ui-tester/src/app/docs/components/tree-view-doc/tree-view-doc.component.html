TreeViewComponent (selector mona-tree-view) renders hierarchical or flat parent-child data as an expandable, keyboard-navigable list of nodes.

Use Tree View to let a consumer browse or act on data that has parent-child relationships, such as a file system, a category tree, or an org chart. The data can be shaped as nested items (hierarchical mode) or as a flat list linked by an id and a parent id (flat mode).

Selection, checking, disabling, expand/collapse control, filtering, and drag-and-drop reordering are all opt-in. None of these behaviors are active until the corresponding attribute directive (monaTreeViewSelectable, monaTreeViewCheckable, monaTreeViewDisable, monaTreeViewExpandable, monaTreeViewFilterable, monaTreeViewDragAndDrop) is applied to the mona-tree-view host element.

Use Grid when data is naturally tabular rather than hierarchical. Use Dropdown List, Combo Box, or Multi Select when the consumer is choosing from a flat collection of options rather than navigating a hierarchy.

textField and either children (hierarchical mode) or idField / parentIdField (flat mode) are required for the tree to render nodes from data.

Consumers interact with the following selectors directly:

  • mona-tree-view – the root component.
  • ng-template[monaTreeViewNodeTemplate] – replaces the default node label with projected content. The template context exposes $implicit (the node's data item) and element (the node's host HTMLElement).
  • mona-tree-view[monaTreeViewSelectable], mona-tree-view[monaTreeViewCheckable], mona-tree-view[monaTreeViewDisable], mona-tree-view[monaTreeViewExpandable], mona-tree-view[monaTreeViewFilterable], mona-tree-view[monaTreeViewDragAndDrop] – attribute directives applied to the mona-tree-view host element to enable each behavior.

Set mode="flat" and provide idField and parentIdField instead of children when the source data is a flat list linked by id references.

Apply monaTreeViewSelectable to track which node is selected. Single mode is the default; pass {{ "{" }} mode: 'multiple' {{ "}" }} to allow multiple selected nodes.

Apply monaTreeViewCheckable to render checkboxes on nodes. Checking a parent also checks its children, and a parent shows an indeterminate state when only some of its children are checked, unless configured otherwise through the directive's options input.

Apply monaTreeViewDisable and pass the keys of the nodes to disable. Disabling a node also disables its descendants by default.

Apply monaTreeViewExpandable and bind expandedKeys two-way to control which nodes are expanded from outside the component.

Apply monaTreeViewFilterable to narrow the visible nodes by text. The tree itself does not render a filter input; wire filter to a consumer-provided control, or place monaTreeViewFilterable without further configuration to enable the tree's own filter region.

Apply monaTreeViewDragAndDrop to let the consumer reorder or reparent nodes by dragging. Dropping a node does not mutate the array passed to data; use the exported moveTreeNode (hierarchical data) or moveFlatTreeNode (flat data) helper inside the nodeDrop handler to compute an updated array and pass it back through data.

Pass a function returning an Observable as children to load a node's children on first expand instead of upfront. Provide hasChildren so unexpanded nodes can render an expand affordance before their children are fetched.

  • nodeSelect, nodeCheck, nodeClick, nodeDragStart, nodeDragEnd, nodeDrop, and filterChange emit an event object with preventDefault() and isDefaultPrevented(). Calling preventDefault() in a handler stops the corresponding state change (selection, check, drag, drop, or filter) from being applied.
  • Switching mode requires providing the matching fields: children for "hierarchical", or idField and parentIdField for "flat". The two field sets are independent inputs and are not cross-validated.
  • TreeViewComponent exposes moveNode(source, target, position) and undoMoveNode(snapshot) for moving nodes in the tree's internal structure directly, without going through data. The nodeDrop event's treeView property refers to the emitting TreeViewComponent instance. Provide ariaLabel when surrounding content does not already identify the tree's purpose.

The tree host is the only tab stop; individual nodes are not part of the page's tab order. Arrow-key navigation moves an internal active node pointer, reflected through aria-activedescendant on the tree host, instead of moving DOM focus between nodes. When the tree regains focus, it restores the previously active node, or activates the first node if none was active yet.

Tree View does not implement ControlValueAccessor or the signal forms FormValueControl / FormCheckboxControl interfaces. It cannot be bound with [(ngModel)], [formControl], formControlName, or [formField]. Track selection or check state through the selectedKeys / checkedKeys models, or the selectionChange / nodeSelect / nodeCheck outputs, exposed by the selectable and checkable directives.
